# Context / Environment — Section Prompt

## Question
In what domain and environment does this invention operate?

## Prompt

Describe the context and environment in which this invention exists and operates. This sets the stage for the patent attorney to understand the field and frame the claims appropriately.

**Novelty Statement:**
{novelty_statement}

**Requirements:**

1. **Domain** — What field or industry does this invention belong to? (e.g., "real-time bidding in digital advertising", "fraud detection in financial services")

2. **System Environment** — Describe the technical environment:
   - What type of system is this part of? (web application, distributed system, embedded system, ML pipeline, etc.)
   - What are the key components it interacts with?
   - What are the runtime characteristics? (real-time, batch, streaming, etc.)

3. **Use Cases** — List the primary use cases where this invention is applied. For each:
   - Who is the user/consumer?
   - What triggers the use of this invention?
   - What is the expected outcome?

4. **Constraints** — What environmental constraints shaped the design?
   - Performance requirements (latency, throughput)
   - Scale requirements (data volume, user count)
   - Resource constraints (memory, compute, network)
   - Regulatory or compliance constraints

5. **Broader Applicability** — Beyond the current use, where else could this invention be applied? (This helps the attorney write broader claims)

**Length:** 300-500 words
