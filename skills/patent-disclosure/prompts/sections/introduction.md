# Introduction — Section Prompt

## Question
What background knowledge is needed to understand this invention?

## Prompt

Provide the technical background a patent attorney (and later a patent examiner) needs to understand the invention. This section educates the reader on the building blocks before the detailed description.

**Novelty Statement:**
{novelty_statement}

**Code Context:**
{code_context}

**Requirements:**

1. **Core Concepts** — Explain each foundational concept the invention builds upon:
   - Define technical terms that will be used throughout the disclosure
   - Explain algorithms, data structures, or techniques that are prerequisites
   - For each concept, explain it at a level appropriate for a technically literate non-specialist

2. **Key Building Blocks** — Describe the existing technologies/techniques this invention leverages:
   - What standard components does it use? (databases, ML frameworks, protocols, etc.)
   - How do these building blocks typically work?
   - What are their known limitations? (this sets up the "problems solved" section)

3. **Relevant Standards or Protocols** — If the invention operates within or extends any standards, name them

4. **Terminology** — Define any domain-specific or invention-specific terms that will be used later. Use a definition list format:
   - **Term**: Definition

**Guidelines:**
- Explain concepts simply but do NOT oversimplify — the patent examiner is technically skilled
- Include brief examples where they aid understanding
- Do NOT describe the invention itself here — only the background needed to understand it
- Do NOT include anything novel — this section is strictly about known prior art and concepts
- Length: 400-700 words
